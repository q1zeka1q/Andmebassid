Piirangut /ограничения
*PRIMARY KEY - -это атрибут (или комбинация атрибутов)1 которыи уникально определяет строку.
Никакие из двух записей таблицы не могут иметь одинаковых значении первичного ключа.
*foreign key - SEOS -связ PK ДРУГОЙ ТАБЛИЦЫ
*NOT NULL - это ограничение (constraint) в базах данных, которое запрещает хранить пустые (NULL) значения в указанном столбце таблицы.
*UNIQUE - это ограничение (constraint) в базе данных, которое гарантирует, что все значения в столбце (или группе столбцов) будут уникальными.
*CHECK - valik


SQL - Structured Query Language
*DDL - (Data Definition Language) – работа со структурой базы

Используется для создания и изменения таблиц.
CREATE – создать новую таблицу или базу данных.
ALTER – изменить таблицу (добавить/удалить столбцы).
DROP – удалить таблицу или базу данных.
TRUNCATE – очистить таблицу (удалить все данные, но оставить структуру).

*DML - (Data Manipulation Language) – работа с данными
Операции для добавления, обновления, удаления и выбора данных.

INSERT – добавить данные.
UPDATE – изменить существующие данные.
DELETE – удалить данные.
SELECT – получить данные.
